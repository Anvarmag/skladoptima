### `apps/web/src/pages/Inventory.tsx`

**Блокировки в таблице остатков:**
- Новый стейт: `locksMap` (map productId → StockLock[]), модальный стейт создания/просмотра блокировок
- `loadAllLocks()` — загружает все блокировки тенанта при монтировании компонента, строит `locksMap`
- В таблице остатков добавлена колонка «Блокировки»:
  - Иконка замка `Lock` рядом со SKU товара при наличии хотя бы одной блокировки; tooltip перечисляет маркетплейс + тип
  - Строка с блокировками подсвечивается фоном `amber-50/30`
  - Отображаются бейджи активных блокировок (только для видимых маркетплейсов)
  - Кнопка-иконка `Unlock`/`Lock` открывает `LockModal`
- **Lock management modal**: список активных блокировок с кнопкой «Снять» (`Trash2`), форма создания новой блокировки (маркетплейс, тип ZERO/FIXED/PAUSED, fixedValue, заметка)
- Оптимистичный UI: блокировка появляется/исчезает немедленно, rollback при ошибке API

**Фильтр видимости каналов:**
- Новый стейт: `visibleMarketplaces`, `channelDropdownOpen`, `visibilityDebounceRef`
- `loadChannelVisibility()` — загружает настройки при монтировании через `GET /inventory/channel-visibility`
- `saveChannelVisibility()` — сохраняет через `PATCH /inventory/channel-visibility` с debounce 500 мс
- Кнопка «Каналы» с иконкой `Eye` над таблицей остатков; dropdown с чекбоксами для WB/OZON
- При снятии галочки колонка скрывается немедленно (бейджи блокировок фильтруются по `visibleMarketplaces`)
- Пустой список не допускается (защита в `toggleMarketplaceVisibility`)

### `apps/web/src/pages/Products.tsx`

**Секция «Связанные артикулы» в модале редактирования:**
- Новые типы: `ChannelMapping`, `MARKETPLACE_LABELS`, `MARKETPLACE_COLORS`
- Новый стейт: `mappings`, `mappingsLoading`, `addMappingOpen`, `addMpMarketplace`, `addMpExtId`, `addMpExtSku`, `addMpError`, `addMpSubmitting`
- `loadMappings(productId)` — загружает маппинги через `GET /catalog/mappings/product/:productId`
- `openEdit()` теперь также вызывает `loadMappings(p.id)` и сбрасывает form-стейт маппингов
- `handleDetachMapping(mappingId)` — оптимистичное удаление + `DELETE /catalog/mappings/:id`, rollback при ошибке
- `handleAddMapping()` — `POST /catalog/mappings/manual`; при 409 показывает сообщение «Этот артикул уже связан с другим товаром (ID: …)»
- UI внутри модала редактирования: список маппингов с кнопкой «Отвязать» (`Unlink`), форма добавления с выбором маркетплейса, полями externalProductId и externalSku
