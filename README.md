# Sklad Optima MVP

Production-ready MVP веб-сервиса для управления складскими остатками.

## Стек технологий
- **Фронтенд:** React, Vite, React Router, TailwindCSS, Axios, Lucide React (иконки).
- **Бэкенд:** NestJS, Prisma, PostgreSQL, JWT (httpOnly cookies), Multer (загрузка файлов).

## Функционал
- 🔐 Защищенная авторизация (логин `admin@example.com` / `admin`).
- 📦 CRUD товаров с загрузкой фотографий и учетом остатков.
- 🧮 Ручная корректировка остатков (инкремент/декремент).
- 🕒 Полная история (Audit Log) всех действий пользователей (создание, удаление, изменение остатков). 
- 💾 Все данные и сессии сохраняются надежно в PostgreSQL (в докере).

## 🚀 Запуск для 개발 (Local Development)

### 1. Подготовка базы данных
Проект требует работающего Docker для развертывания базы данных (PostgreSQL).

```bash
docker-compose up -d
```

### 2. Настройка базы
Перейдите в папку бэкенда и создайте таблицы:

```bash
cd apps/api
npm install
npx prisma migrate dev --name init
```

*При первом запуске автоматически создастся админ: `admin@example.com`, пароль: `admin`.*

### 3. Запуск всего проекта (Бэкенд + Фронтенд)
Находясь в **корневой** папке проекта (`Skladoptima`), выполните команду, которая одновременно запустит NestJS (порт 3000) и React Vite (порт 5173):

```bash
npm run dev
```

Откройте в браузере: **[http://localhost:5173](http://localhost:5173)**

## 🌍 Руководство по деплою на VPS (Ubuntu)

Когда проект будет готов к релизу:

1. **База данных:** На сервере запускаем тот же `docker-compose.yml` (Postgres).
2. **Бэкенд:** 
   - `cd apps/api && npm run build`
   - Запускаем через PM2: `pm2 start dist/main.js --name "sklad-api"`
3. **Фронтенд:**
   - `cd apps/web && npm run build`
   - Статичные файлы из папки `dist` будут раздаваться через Nginx.
4. **Nginx Reverse Proxy:**
   - Настраиваем домен на сервер.
   - `/api/*` -> `http://localhost:3000/api`
   - `/uploads/*` -> `http://localhost:3000/uploads`
   - `/` -> раздача статики React (`try_files $uri /index.html;`)

*В будущем здесь же будет добавлена логика синхронизации (Marketplace API)*.
