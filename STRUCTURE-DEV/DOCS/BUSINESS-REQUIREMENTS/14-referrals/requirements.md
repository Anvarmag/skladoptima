1. Описание раздела

Раздел определяет:

реферальную механику привлечения новых клиентов
учет успешных рефералов
бонусный счет реферера
использование бонусов для оплаты подписки
промокоды и скидки
ограничения и антифрод-правила

На MVP реферальная система и промокоды являются маркетинговым и retention-слоем поверх биллинга и подписок.

2. Целевая аудитория
Роль	Действия доступны
Primary Owner	Получение реферальной ссылки, просмотр приглашенных, просмотр бонусов, использование бонусного счета
Новый пользователь / новый tenant	Переход по реферальной ссылке, использование промокода при оплате
SUPPORT_ADMIN	Управление промокодами и служебное сопровождение системы
3. Бизнес-цели
Стимулировать привлечение новых клиентов через действующих пользователей.
Поддержать маркетинговые скидочные кампании через промокоды.
Повысить конверсию в первую оплату после trial.
Дать рефереру понятный и полезный бонус без сложной payout-системы.
Создать основу для дальнейшего роста referral / promo механик.
4. Базовые бизнес-правила
Реферальная ссылка выдается каждому Primary Owner.
Реферером на MVP считается Primary Owner, а не tenant.
У Primary Owner есть единый внутренний бонусный счет.
Бонусный счет используется только для оплаты подписок tenant, где пользователь является Primary Owner.
Бонусы не выводятся в деньги на MVP.
Успешный referral засчитывается после первой успешной оплаты приглашенного tenant.
Бонус за referral начисляется один раз.
Модель процента с каждой будущей оплаты не входит в MVP.
Промокоды и реферальная система являются разными сущностями.
Промокоды могут использоваться не только на первую оплату, а по условиям конкретного промокода.
Self-referral запрещен.
Скидки и бонусы на MVP не должны превращаться в полноценный денежный кошелек или payout-систему.
5. Основные сущности
5.1 Referral Link

Персональная ссылка Primary Owner для привлечения новых клиентов.

5.2 Referral Attribution

Фиксация факта, что новый tenant пришел по конкретной реферальной ссылке.

5.3 Referral Bonus

Бонус, начисляемый за успешного реферала.

5.4 Bonus Balance

Внутренний бонусный счет Primary Owner, который можно использовать для оплаты подписки.

5.5 Promo Code

Маркетинговый код скидки с условиями применения.

6. Пользовательские сценарии
US-01

Как Primary Owner, я хочу получить реферальную ссылку, чтобы приглашать новых клиентов.

US-02

Как Primary Owner, я хочу видеть количество приглашенных пользователей и оплаченных рефералов, чтобы понимать эффективность моих приглашений.

US-03

Как Primary Owner, я хочу видеть свой бонусный баланс, чтобы использовать его для оплаты подписки.

US-04

Как новый пользователь, я хочу перейти по реферальной ссылке, чтобы зарегистрироваться в системе.

US-05

Как новый пользователь, я хочу использовать промокод при оплате, чтобы получить скидку.

US-06

Как система, я хочу начислить бонус рефереру после первой успешной оплаты нового tenant.

9. Бонусный счет
Бонусный счет является внутренней учетной сущностью системы.
Бонусный счет не является банковским или выводимым денежным кошельком.
Бонусный баланс может использоваться только внутри SkladOptima.
Бонусный баланс можно применять к оплате подписок tenant, где владелец баланса является Primary Owner.
Бонусный баланс должен отображаться в интерфейсе пользователя.
Начисления и списания бонусного баланса должны логироваться.
10. Промокоды
Типы промокодов
PERCENT - процентная скидка
FIXED - фиксированная сумма скидки
Ограничения промокодов

Каждый промокод может иметь:

ограничение по количеству использований (maxUses)
срок действия (expiresAt)
дополнительные условия применения
Применение
промокоды могут использоваться не только на первую оплату
правила применения определяются настройками конкретного промокода
система должна поддерживать маркетинговые, акционные и retention-сценарии применения промокодов

12. Anti-fraud и ограничения
Self-referral запрещен.
Один и тот же пользователь не должен получать реферальный бонус за самого себя.
Повторное начисление бонуса за один и тот же tenant не допускается.
Успешный referral должен засчитываться только один раз.
Начисление бонусов должно быть идемпотентным.
Логика начисления не должна срабатывать повторно при повторной обработке одной и той же оплаты.
13. История и аудит

Система должна сохранять историю как минимум по следующим событиям:

реферальная ссылка создана / активна
новый referral атрибутирован
referral стал успешным
бонус начислен
бонус использован
промокод применен
промокод отклонен
self-referral заблокирован
14. Ограничения и правила
Реферальная система и промокоды - разные сущности и должны учитываться отдельно.
Бонусный баланс не является денежным кошельком с выводом средств.
Бонусы на MVP используются только для оплаты подписки.
Процент с каждой будущей оплаты не поддерживается на MVP.
Промокоды могут иметь разные правила применения в зависимости от их конфигурации.
Условия использования промокодов определяются настройками самого промокода, а не одним глобальным правилом для всей системы.
15. Что не входит в этот раздел

Чтобы не размывать границы, в этот раздел не входят:

денежные выплаты на карту или счет
процент с каждой будущей оплаты
многоуровневая реферальная система
полноценный бонусный кошелек с выводом средств
cashback
сложные комбинированные discount stacks
affiliate payout history как отдельный финансовый модуль
сложный anti-fraud scoring
16. Комментарии на будущее
В будущем можно добавить процент от повторных оплат.
В будущем можно расширить бонусный баланс до более гибкой модели использования.
В будущем можно добавить более сложные промо- и referral-кампании.
В будущем можно добавить Telegram- и иные каналы уведомлений по referral-событиям.
В будущем можно добавить более детальную аналитику по конверсии реферальной воронки.
