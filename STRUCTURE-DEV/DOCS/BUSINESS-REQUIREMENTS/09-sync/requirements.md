## 1. Описание раздела

Фоновая синхронизация остатков и заказов между SkladOptima и WB/Ozon. Автоматический цикл каждые 60 сек.

---

## 2. Целевая аудитория

| Роль | Действия доступны |
|------|------------------|
| OWNER / ADMIN | Ручной запуск, просмотр логов |
| MANAGER | Просмотр статуса синхронизации |

## 4. Пользовательские сценарии (User Stories)

**US-01:** Как OWNER, я хочу видеть когда последний раз прошла синхронизация.

**US-02:** Как ADMIN, я хочу запустить синхронизацию вручную, если нужно обновить данные срочно.

**US-03:** Как система, я должна синхронизировать остатки автоматически каждые 60 сек.

---

## 5. Функциональные требования

### Обязательные (Must Have)

- [ ] FR-01: Автоматический цикл sync каждые 60 сек
- [ ] FR-02: pullFromWb — стянуть FBS остатки
- [ ] FR-03: pullFromOzon — стянуть FBS+FBO остатки
- [ ] FR-04: pushToWb / pushToOzon — передать обновлённые остатки
- [ ] FR-05: processWbOrders / processOzonOrders — списать остатки по заказам
- [ ] FR-06: Ping-pong protection (cooldown 2 мин после push)
- [ ] FR-07: Ручной запуск из UI

### Желательные (Should Have)

- [ ] FR-10: Логи синхронизации (MarketplaceSyncRun)
- [ ] FR-11: Уведомление при ошибке синхронизации
- [ ] FR-12: Выделенный Worker процесс

---

## 7. Бизнес-правила

- После push на маркетплейс — cooldown 2 минуты на pull этого товара.
- FBO заказы Ozon — не списывают из total (маркетплейс сам отгружает).
- WB FBO — пока не поддерживается.
- Дедупликация заказов по marketplaceOrderId.
