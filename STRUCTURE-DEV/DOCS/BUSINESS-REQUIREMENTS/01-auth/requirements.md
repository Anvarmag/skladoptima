## 1. Описание раздела

Управление входом и выходом пользователей, хранение сессий, защита маршрутов, поддержка Telegram WebApp логина.

---

## 2. Целевая аудитория

| Роль | Действия доступны |
|------|------------------|
| Все пользователи | Регистрация, логин, логаут, смена пароля |
| OWNER | Управление настройками безопасности тенанта |

## 4. Пользовательские сценарии (User Stories)

**US-01:** Как новый пользователь, я хочу зарегистрироваться по email, чтобы создать аккаунт.

**US-02:** Как пользователь Telegram, я хочу войти через Telegram WebApp, чтобы не вводить пароль.

**US-03:** Как пользователь, я хочу восстановить пароль, чтобы не потерять доступ.

---

## 5. Функциональные требования

### Обязательные (Must Have)

- [ ] FR-01: Регистрация по email + пароль
- [ ] FR-02: Логин по email + пароль
- [ ] FR-03: JWT в httpOnly cookie, срок 7 дней
- [ ] FR-04: Логаут (инвалидация cookie)
- [ ] FR-05: Telegram WebApp авторизация
- [ ] FR-06: JWT содержит userId, tenantId, membershipId, role

### Желательные (Should Have)

- [ ] FR-10: Смена пароля
- [ ] FR-11: Email-верификация при регистрации

### На будущее (Could Have)

- [ ] FR-20: 2FA
- [ ] FR-21: OAuth (Google)

---

## 6. Нефункциональные требования

| Требование | Значение |
|-----------|---------|
| Безопасность | httpOnly cookie, bcrypt для паролей, Helmet |
| Сессия | 7 дней, обновляемый токен |
| Защита от брутфорса | Rate limiting на /auth/login |

---

## 7. Бизнес-правила и ограничения

- Один email = один User (не один Tenant).
- User может быть членом нескольких Tenant.
- JWT НЕ содержит storeId (устаревший паттерн).
